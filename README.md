# Grupo7_Campus-Lite
---

# 1. Descripción del Proyecto

Campus Lite es un sistema académico desarrollado en Java que permite gestionar estudiantes, cursos, inscripciones y reportes académicos mediante una interfaz gráfica desarrollada con Java Swing.

El objetivo principal de este proyecto fue aplicar los conocimientos vistos durante el curso, especialmente temas relacionados con Programación Orientada a Objetos, persistencia de datos, interfaces gráficas y trabajo colaborativo utilizando herramientas como GitHub y Trello.

Además, el sistema fue diseñado para facilitar el control y administración de información académica de manera sencilla y organizada.

# 5. Desarrollo del Proyecto

## 5.1 Semana 1

Durante la primera semana no se comenzó directamente con código, sino que primero se analizó el proyecto y se investigó qué era lo que se debía realizar.

El equipo se organizó para comprender mejor los requerimientos del sistema y las funcionalidades que debía incluir. También se investigó sobre cómo trabajar colaborativamente utilizando GitHub y Trello.

Además, se realizó un boceto inicial de la interfaz gráfica utilizando Figma para tener una idea visual de cómo se vería el sistema antes de comenzar la programación.

En esta etapa se discutieron ideas, estructura del proyecto y organización general del trabajo.

---

## 5.2 Semana 2

En la segunda semana ya se comenzó con el desarrollo del código.

Principalmente se trabajó en la lógica del sistema utilizando Programación Orientada a Objetos. Se desarrollaron las clases del paquete `domain`, definiendo atributos, métodos, constructores y relaciones entre las diferentes entidades del proyecto.

También se trabajó en los paquetes `service` y `persistence`, encargados de manejar la lógica del programa y el almacenamiento de información utilizando archivos TXT y CSV.

Durante esta semana se implementaron funcionalidades importantes como:

* Registro de estudiantes
* Registro de cursos
* Inscripciones
* Eliminación de registros
* Búsquedas
* Persistencia de datos

---

## 5.3 Semana 3

En la tercera semana se trabajó principalmente la interfaz gráfica utilizando Java Swing.

Se desarrollaron las ventanas del sistema, formularios, botones, tablas y reportes académicos para que el programa fuera más amigable visualmente y fácil de utilizar.

Además, se realizaron pruebas generales del sistema para detectar errores y solucionar problemas relacionados con la integración entre la lógica, persistencia e interfaz gráfica.

Finalmente, se hicieron correcciones finales, mejoras visuales y documentación del proyecto.

---

# 6. Estructura del Proyecto

## 6.1 Package domain

En este paquete se encuentran las clases principales del sistema, tales como:

* Student
* Course
* Registration
* Evaluation

Aquí se aplicaron conceptos importantes de Programación Orientada a Objetos como encapsulamiento, constructores, atributos y métodos.

---

## 6.2 Package service

Este paquete contiene la lógica principal del sistema y permite realizar operaciones como agregar, eliminar, buscar y administrar información.

---

## 6.3 Package persistence

Se encarga del manejo y almacenamiento de datos utilizando archivos TXT y CSV para mantener la información guardada.

---

## 6.4 Package ui

Contiene toda la interfaz gráfica desarrollada con Java Swing.

En este paquete se encuentran las diferentes ventanas del sistema y la interacción visual con el usuario.

---

# 7. Funcionalidades del Sistema

El sistema permite realizar las siguientes acciones:

* Registrar estudiantes
* Registrar cursos
* Realizar inscripciones
* Buscar información
* Eliminar registros
* Generar reportes académicos
* Guardar información automáticamente

# 10. Problemas Encontrados

Durante el desarrollo del proyecto surgieron algunos problemas relacionados con la persistencia de archivos, la integración entre ventanas y la conexión de clases dentro del sistema.

Además, debido a que varios integrantes del grupo también tenían responsabilidades laborales y poco tiempo disponible, en algunos momentos fue complicado coordinar horarios y avanzar al mismo ritmo.

Sin embargo, mediante pruebas constantes, revisiones del código, comunicación entre los integrantes y trabajo en equipo, fue posible solucionar los errores encontrados y lograr que el sistema funcionara correctamente de manera integrada y estable.

---

# 11. Conclusiones

Este proyecto permitió aplicar los conocimientos aprendidos durante el curso, especialmente temas relacionados con Programación Orientada a Objetos, interfaces gráficas y persistencia de datos.

Además, ayudó a mejorar el trabajo en equipo, la organización de tareas y el uso de herramientas colaborativas utilizadas en proyectos reales de desarrollo de software.

Finalmente, se logró desarrollar un sistema funcional y organizado capaz de administrar información académica mediante una interfaz gráfica amigable y fácil de utilizar.
